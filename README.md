# OpenCart Manual Testing Project

## Project Overview

This project demonstrates end-to-end Manual Testing of the OpenCart E-Commerce Web Application. The objective was to validate core functionalities, identify defects, and prepare comprehensive testing documentation following industry-standard QA practices.

## Modules Tested

* Registration
* Login
* Search
* Cart

## Testing Activities Performed

* Requirement Analysis
* Test Scenario Creation
* Test Case Design
* Functional Testing
* Negative Testing
* Boundary Value Analysis (BVA)
* Exploratory Testing
* Defect Reporting

## Project Deliverables

* Requirements Document
* Test Scenarios
* 100+ Test Cases
* Test Execution Report
* Bug Report
* Screenshots

## Defects Identified

### BUG-001

Search functionality returns Error 520 when search input exceeds approximately 700 words.

### BUG-002

Application displays a success message when alphabetic characters are entered in the quantity field, but the product is not added to the cart.
